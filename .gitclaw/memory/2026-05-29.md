# 2026-05-29

- GitClaw v0 issue/comment assistant works in the main repository.
- `openai/gpt-5-mini` works through GitHub Models in Actions.
- Issue backups are committed to the `gitclaw-backups` branch after successful turns.
- Heartbeat should be implemented as a scheduled/manual GitHub Actions workflow,
  not a server. It scans `gitclaw:heartbeat` issues and uses same-slot
  idempotency.
- Read-only tools should include bounded search as `gitclaw.search_files`, with
  matching lines inserted as auditable prompt context before model inference.
- Issue-chat turns should manage lightweight status labels:
  `gitclaw:running`, `gitclaw:done`, and `gitclaw:error`.
- Failed started turns should post a bounded `gitclaw:error` diagnostic comment
  that points to the Actions run without leaking issue tokens or raw provider
  response bodies.
- Prompt assembly should enforce byte budgets before model calls: preserve the
  original issue and recent transcript tail, truncate large bodies visibly, and
  cap the final prompt.
- Prompt artifacts should be opt-in, redacted before upload, and stored as
  GitHub Actions artifacts rather than issue comments or logs.
- Write intent should be detected deterministically in read-only mode, labeled
  `gitclaw:write-requested`, and answered as a proposal rather than as a
  completed file change.
- Main issue handling can be woken with `workflow_dispatch` using
  `issue_number` plus a stable `dispatch_id`; this is the bridge-friendly path
  for scheduled Telegram/Slack pollers because `GITHUB_TOKEN`-authored comments
  should not be expected to recursively trigger `issue_comment` workflows.
- Mirrored channel comments should start with a hidden
  `gitclaw:channel-message` marker. GitClaw reconstructs those comments as
  untrusted user transcript messages even if a bridge posted them as
  `github-actions[bot]`.
- Generic channel ingest should be a workflow-dispatchable path: normalized
  channel/thread/message fields create or reuse a canonical
  `gitclaw:channel-thread` issue, post a `gitclaw:channel-message` comment,
  then dispatch the main handler with `<channel>-<message_id>`.
- Proactive usefulness belongs in the spec as scheduled GitHub Actions
  workflows that create or reuse their own `gitclaw:proactive-run` issues and
  then dispatch the normal handler. Do not silently let the agent install new
  cron workflows without review.
- Proactive enqueue is now the first implemented primitive: it creates/reuses
  one issue per proactive `name + slot`, writes a `gitclaw:proactive-run`
  marker, applies `gitclaw:proactive`, and dispatches the normal handler with
  `proactive-<name>-<slot>`.
- GitHub Models can return bursty `429` responses during dense live E2E runs;
  the OpenAI-compatible client now retries transient `429`, timeout, and `5xx`
  failures and uses a bounded HTTP timeout before posting the safe
  provider-failure comment. `Retry-After` delays are capped so rate-limit
  cooldowns cannot park an Actions worker indefinitely.
- `@gitclaw /context` is the deterministic context-inspection command. It
  assembles normal repo context, posts a `model="gitclaw/context"` report with
  context file, skill, and tool-output names/sizes, and avoids a model call.
- `@gitclaw /memory` is the deterministic memory-surface audit command. It
  reports `.gitclaw/MEMORY.md`, dated memory note counts, loaded/omitted note
  counts, canonical filename hygiene, and hashes without dumping memory bodies
  or allowing hidden memory writes.
- `@gitclaw /skills` is the deterministic local skill inventory command. It
  posts a `model="gitclaw/skills"` report with available and selected skill
  metadata, including hashes, frontmatter/description presence, and declared
  env/bin requirement counts, validation status, duplicate-name counts,
  invalid-name counts, and folder/name mismatch counts, without dumping full
  `SKILL.md` bodies or calling a model.
- `gitclaw skills validate` prints the same body-free local skill validation
  findings outside an issue workflow.
- The `gitclaw-backups` branch now keeps a repo-scoped backup index beside raw
  issue JSON backups: `.gitclaw/backups/<owner>__<repo>/index.json` and
  `README.md`, with issue metadata and counts but no raw message bodies.
- `@gitclaw /soul` is the deterministic high-authority context audit command.
  It posts a `model="gitclaw/soul"` report listing loaded soul, identity, user,
  memory, heartbeat, and dated memory files with sizes and hashes, without
  dumping file bodies or calling a model. The report now includes soul
  validation status, required-file counts, memory-note counts, noncanonical
  memory-note counts, and body-free findings.
- `gitclaw soul validate` prints the same body-free local validation findings
  outside an issue workflow.
- `@gitclaw /tools` is the deterministic tool-surface audit command. It posts a
  `model="gitclaw/tools"` report listing available read-only tool contracts and
  active tool output metadata with inputs, sizes, and hashes, without dumping
  tool output bodies or calling a model. The report now includes tool
  validation status, contract counts, active-output counts, unknown-output
  counts, unsafe-contract counts, over-limit output counts, and body-free
  findings, plus missing-guidance and duplicate-contract counts.
- `gitclaw tools validate` prints the same body-free local tool validation
  findings outside an issue workflow.
- `@gitclaw /prompt` is the deterministic prompt-budget audit command. It posts
  a `model="gitclaw/prompt"` report with provider/model metadata, final prompt
  size/hash, transcript omission/truncation counts, selected context/skill
  files, and active tool output metadata, without dumping prompt or body text.
- `@gitclaw /policy` is the deterministic preflight/policy audit command. It
  posts a `model="gitclaw/policy"` report with trusted actor state, managed
  labels, write-request detection, expected workflow permissions, and
  `gitclaw.policy` metadata, without dumping issue bodies or calling a model.
- `@gitclaw /session` is the deterministic issue-session audit command. It
  posts a `model="gitclaw/session"` report with raw comment counts,
  reconstructed transcript counts, marker counts, per-message metadata, and
  message hashes, without dumping transcript bodies or calling a model.
- `@gitclaw /backup` is the deterministic backup-path audit command. It posts a
  `model="gitclaw/backup"` report with the backup branch, issue JSON path,
  repo index/README paths, schema version, and current transcript counts, then
  the normal post-turn backup job writes the canonical raw backup.
- `gitclaw backup verify --root .gitclaw/backups --repo <owner/repo>` validates
  the fetched backup branch: repo index, README, traversal-safe canonical issue
  JSON paths, schema/count/timestamp consistency, and absence of unindexed issue
  backup files.
- `gitclaw backup manifest --root .gitclaw/backups --repo <owner/repo>
  --issue <n>` renders a file-level backup manifest with control-file and
  issue-payload hashes, counts, and byte sizes, without dumping raw bodies.
- `gitclaw backup stats --root .gitclaw/backups --repo <owner/repo>` verifies
  and summarizes a fetched backup branch with aggregate issue/comment/message
  counts, event type counts, latest backup metadata, and payload bytes, without
  dumping raw titles or bodies.
- `gitclaw backup export-jsonl --root .gitclaw/backups --repo <owner/repo>
  --issue <n>` exports backed-up reconstructed transcript messages as raw JSONL
  recovery records. It is intentionally local CLI output, not an issue-visible
  diagnostic.
- `gitclaw backup restore-plan --root .gitclaw/backups --repo <owner/repo>
  --issue <n>` renders a non-mutating restore dry-run with backup metadata,
  counts, and body hashes, without calling GitHub or dumping raw bodies.
- `@gitclaw /proactive` is the deterministic proactive-surface audit command.
  It reports the proactive workflow, `workflow_dispatch`/`schedule` trigger
  presence, prompt file metadata, labels, and enqueue idempotency contract
  without dumping prompt or issue bodies.
- The generic proactive workflow now has a weekly scheduled path plus a safe
  `.gitclaw/proactive/repo-hygiene.md` default prompt so proactive usefulness is
  available without a daemon.
- `gitclaw proactive init --name <name> --cron <cron>` generates a proactive
  prompt file and scheduled workflow as ordinary repo files. It supports
  `--dry-run`, refuses differing overwrites without `--force`, and reports
  hashes/status without dumping prompt bodies.
- Deterministic slash commands can be triggered by a line in the issue body
  that starts with `@gitclaw /...`, which lets proactive issue prompts request
  body-free reports without forcing a model call.
- `@gitclaw /models` is the deterministic model-provider audit command. It
  reports GitHub Models/OpenAI-compatible provider family, selected model, token
  source name, timeout, retry attempts, and bounded exponential backoff settings
  without making a model call or exposing token values.
- Model-backed Actions runs now use a stronger live retry budget:
  6 attempts, 75 second request timeout, 10 second base retry delay, and
  90 second maximum retry delay. Source defaults are 5 attempts, 60 seconds,
  5 seconds, and 60 seconds.
- Checked-in workflows now use Node 24-compatible first-party actions:
  `actions/checkout@v5`, `actions/setup-go@v6`, and
  `actions/upload-artifact@v6`, avoiding GitHub's 2026 Node.js 20 action
  deprecation warnings.
- `@gitclaw /config` is the deterministic control-plane audit command. It
  reports effective defaults/env config, trigger settings, managed labels,
  trusted associations, prompt budgets, slash commands, and workflow/config file
  metadata without exposing config, workflow, issue, or comment bodies.
- `@gitclaw /help` and `@gitclaw /commands` are deterministic command catalog
  aliases. They report canonical slash commands, aliases, body-free model
  marker names, categories, summaries, and local CLI helpers. `gitclaw commands`
  prints the same catalog locally.
- `@gitclaw /skills info <name>` and `gitclaw skills info <name>` render a
  focused skill metadata report with path, hash, selection state, requirement
  names/counts, and validation findings, without dumping full `SKILL.md`
  bodies or secret values.
- `@gitclaw /channels` is the deterministic channel bridge audit command. It
  reports channel-ingest workflow metadata, dispatch/permission flags,
  provider keys, channel-thread status, and mirrored channel-message counts
  without exposing issue, command, workflow, or channel message bodies.
- `.gitclaw/config.yml` is now a read-only, schema-checked repo config file.
  GitClaw loads built-in defaults, then repo config, then environment overrides;
  unknown config keys are rejected so secrets and accidental OpenClaw-shaped
  fields do not silently become runtime policy.
- `@gitclaw /doctor` is the deterministic cold health-check command. It reports
  config validation, workflow presence, context file metadata, skills, memory
  notes, proactive prompts, model host metadata, managed-label counts, and
  skill/soul/tool validation rollups without dumping file, prompt, issue, or
  comment bodies.
